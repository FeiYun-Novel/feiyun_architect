# 任务卡片模板（七字段）

复制本骨架，填好后存到 `cards/YYYY-MM-DD-主题.md`。

```markdown
---
name: <任务名>
created: YYYY-MM-DD
type: 普通任务 | goal | loop | schedule | workflow/混合
status: running | done | stopped
---

1. 目标：要达成什么
2. 完成标准：怎么算完成（尽量可量化：测试数/分数/数量/用户点头）
3. 触发方式：手动一次 / 每 N 分钟 / 云端定时 / 事件
4. 验证方法：每轮怎么自检（命令、截图、比对……）
5. 边界（不能动）：不许碰的文件/系统/操作（默认继承：不 push；其余按你自己的全局边界填，如某配置不动、某目录只读）
6. 停止条件：达标即停 + 最多 N 轮 / 超时 / 用户喊停
7. 失败与回滚：卡住 N 轮怎么办；备份放在哪、恢复命令原文（照抄即可回到开跑前）
8. 模型 + effort：每一步建议的模型和 effort 档位，及理由（省 token 优先、质量兜底）

## 交付的 prompt（原样存档，方便复跑）

（把交付给用户的完整命令/prompt 原文贴在这里）

## 复盘（结束后回填）

- 卡在哪、超没超预期、下次模板要改什么
```

## 填写规则

- 七个字段一个都不能空着启动；能推断的填默认值并标注「(默认)」。
- 「完成标准」优先量化；实在只能「用户满意」，就在「验证方法」里写清每轮给用户看什么来判断。
- 循环类任务的「停止条件」必须含轮数或时间上限。
